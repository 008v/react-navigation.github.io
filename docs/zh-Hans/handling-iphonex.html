适配 iPhone X

默认情况下，React Navigation 会帮助确保您的应用程序在iPhoneX上正确显示。 它通过在可能与传感器集群（“齐刘海”）或 Home 主页指示器交互的UI元素内部使用SafeAreaView来实现。

隐藏/自定义导航栏或选项卡栏

但是，如果你要覆盖默认导航栏，请务必确保你的用户界面不会对这些硬件元素中的任何一个造成干扰。

例如，如果我为header或tabBarComponent不提供任何内容，则不会渲染任何内容

const Tabs = createBottomTabNavigator({
  ...
}, {
  tabBarComponent: () => null,
});

export default createStackNavigator({
  ...
}, {
  headerMode: 'none',
});

要解决这个问题，你可以将你的内容包装在一个SafeAreaView中，它可以从react-navigation导入。

import { SafeAreaView } from 'react-navigation';

class App extends Component {
  render() {
    return (
      <SafeAreaView style={styles.container}>
        <Text style={styles.paragraph}>
          This is top text.
        </Text>
        <Text style={styles.paragraph}>
          This is bottom text.
        </Text>
      </SafeAreaView>
    );
  }
}

这将检测应用程序是否在 iPhone X 上运行，如果是，确保内容不会被任何硬件元素遮挡。

Landscape Mode（横屏模式）

即使你的应用程序在横屏模式下使用默认的 navigation bar 和 tab bar，确保内容不会被齐刘海遮挡也很重要。

要解决此问题，您可以再次将您的内容包装在SafeAreaView中。 这不会与竖屏模式下的 navigation bar 和 tab bar 的默认行为相冲突。

总之，请在使用 React Navigation 导航器注册的页面上使用SafeAreaView组件。

使用 forceInset 获得更多控制

在某些情况下，你可能需要更多控制应用哪些填充。 例如，你可以通过将forceInset 这个 prop 传递给SafeAreaView来移除底部填充。

<SafeAreaView style={styles.container} forceInset={{ bottom: 'never' }}>
  <Text style={styles.paragraph}>
    This is top text.
  </Text>
  <Text style={styles.paragraph}>
    This is bottom text.
  </Text>
</SafeAreaView>

forceInset使用 key（取值：top | bottom | left | right | vertical | horizontal）和value（取值：'always' | 'never'）组成一个对象 或者，您可以通过传递一个整数来完全重写填充。
